### What I do

I build and operate the layer between the internet and production: high-scale **edge** (Varnish, WebAssembly), production **Kubernetes** platforms, and the **reliability** and **security** practices that keep them trustworthy. At OTTO I'm the lead architect of the edge and compute platform behind one of Germany's largest e-commerce sites.

Most of my focus now is **sovereign, private AI infrastructure** — running open LLMs, RAG, and coding agents on your own Kubernetes, with the governance and auditability that regulated environments (EU AI Act, GDPR) actually require.

### 🚀 Featured work

**[private-ai-platform-kit](https://github.com/RamazanKara/private-ai-platform-kit)** — Kubernetes-native reference platform for private LLMs, RAG, and coding agents: vLLM/Ollama serving, Qdrant, GPU autoscaling with KEDA, Argo CD GitOps, policy-as-code with Kyverno, OpenTelemetry tracing, and full supply-chain attestation (SBOM, Sigstore, SLSA).

**[kube-shield](https://github.com/RamazanKara/kube-shield)** — Kubernetes security posture manager in Go. Scans clusters for CIS, RBAC, network-policy, and secrets risks; emits SARIF for CI gating; ships signed, SBOM'd releases.

**[vmod-wasm](https://github.com/RamazanKara/vmod-wasm)** — Varnish 9 module that runs WebAssembly / Proxy-Wasm filters at the edge via Wasmtime — sandboxed custom logic right in the request path.

**[restore-drill](https://github.com/RamazanKara/restore-drill)** — Automated disaster-recovery restore verification for PostgreSQL, MySQL/MariaDB, and Redis, with Prometheus metrics and compliance evidence.

**[otto-de/prometheus_varnish_exporter](https://github.com/otto-de/prometheus_varnish_exporter)** — Maintainer · Prometheus exporter for Varnish cache metrics.
